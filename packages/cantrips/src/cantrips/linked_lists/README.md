# Linked Lists

## Overview

Linked lists test pointer manipulation and in-place algorithms. Common patterns
include reversal, cycle detection, and fast/slow pointers.

## Core Concepts

### Concept 1: Fast & Slow Pointers

**What it is**: Two pointers moving at different speeds **When to use**: Finding
middle, cycle detection **Time complexity**: O(n)

### Concept 2: In-place Reversal

**What it is**: Reversing links without extra space **When to use**: "Reverse linked
list", reordering **Time complexity**: O(n), O(1) space

### Concept 3: Dummy Head

**What it is**: Placeholder node before actual head **When to use**: When head might
change, simplifies edge cases **Time complexity**: O(1) extra space

## Common Patterns

### Pattern 1: Cycle Detection (Floyd's)

**Recognition**: "Detect cycle", "find loop" **Approach**: Fast pointer moves 2x,
slow moves 1x **Example problems**: Linked List Cycle

### Pattern 2: Merge Lists

**Recognition**: "Merge sorted lists", "combine" **Approach**: Two pointers comparing
values **Example problems**: Merge Two Sorted Lists

### Pattern 3: In-place Reversal

**Recognition**: "Reverse", "reverse between" **Approach**: Track prev, current, next
pointers **Example problems**: Reverse Linked List
