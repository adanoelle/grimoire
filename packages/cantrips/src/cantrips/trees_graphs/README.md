# Trees & Graphs

## Overview

Trees and graphs are crucial for representing hierarchical and networked data. Master
DFS, BFS, and tree traversals.

## Core Concepts

### Concept 1: DFS (Depth-First Search)

**What it is**: Explore as deep as possible before backtracking **When to use**: Path
finding, tree traversal, connected components **Time complexity**: O(V + E)

### Concept 2: BFS (Breadth-First Search)

**What it is**: Explore level by level **When to use**: Shortest path, level-order
traversal **Time complexity**: O(V + E)

### Concept 3: Tree Traversals

**What it is**: Inorder, preorder, postorder **When to use**: Different orderings of
tree nodes **Time complexity**: O(n)

## Common Patterns

### Pattern 1: Recursive DFS

**Recognition**: "All paths", "sum to target" **Approach**: Recursively explore
children **Example problems**: Path Sum, Max Depth

### Pattern 2: Level-order BFS

**Recognition**: "Level-order", "by level" **Approach**: Queue with level tracking
**Example problems**: Binary Tree Level Order

### Pattern 3: Graph DFS/BFS

**Recognition**: "Connected components", "shortest path" **Approach**: Track visited
nodes **Example problems**: Number of Islands
