# Stacks & Queues

## Overview

Stacks (LIFO) and queues (FIFO) are foundational for many algorithms. Key patterns
include monotonic stacks, BFS with queues, and expression evaluation.

## Core Concepts

### Concept 1: Stack (LIFO)

**What it is**: Last in, first out data structure **When to use**: Matching pairs,
backtracking, DFS **Time complexity**: O(1) push/pop

### Concept 2: Queue (FIFO)

**What it is**: First in, first out data structure **When to use**: BFS, level-order
traversal, scheduling **Time complexity**: O(1) enqueue/dequeue (with deque)

### Concept 3: Monotonic Stack

**What it is**: Stack maintaining increasing/decreasing order **When to use**: Next
greater/smaller element, spanning problems **Time complexity**: O(n) overall

## Common Patterns

### Pattern 1: Matching Pairs

**Recognition**: "Valid parentheses", "balanced" **Approach**: Push opening, pop on
closing **Example problems**: Valid Parentheses

### Pattern 2: Monotonic Stack

**Recognition**: "Next greater", "next warmer" **Approach**: Maintain
increasing/decreasing stack **Example problems**: Next Greater Element, Daily
Temperatures

### Pattern 3: BFS with Queue

**Recognition**: "Level-order", "shortest path" **Approach**: Queue for
level-by-level traversal **Example problems**: Binary Tree Level Order
